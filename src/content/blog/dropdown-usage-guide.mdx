# Dropdown Component - Quick Usage Guide

Dynamic dropdown component for Astro MDX that switches content based on user selection. Perfect for language translations, code examples, and documentation.

## üì¶ Import the Components

```jsx
import Dropdown, { DropdownOption, DropdownRef } from "@/react/Dropdown/Dropdown.jsx";
// or use backward compatible import
import Dropdown, { Option, DropdownRef } from "@/react/Dropdown/Dropdown.jsx";
```
or with relative path:

```jsx
import Dropdown, { DropdownOption, DropdownRef } from "../../react/Dropdown/Dropdown.jsx";
```

## ‚öôÔ∏è Component Props

### Dropdown Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultValue` | string | First option | Which option to show initially |
| `placeholder` | string | "Select an option..." | Text shown before selection |
| `stripCss` | boolean | false | Remove background styling |
| `client:load` | directive | - | **Required** for Astro |

### DropdownOption Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `value` | string | Yes | Unique identifier |
| `label` | string | No | Display text (auto-generated if omitted) |
| `children` | content | Yes | Content to display |

### How to Use References

**Step 1:** Define dropdown with reference pattern `:::drop <id>`

```jsx

  <DropdownOption value=":::drop javascript-ref" label="JavaScript" />
  <DropdownOption value=":::drop python-ref" label="Python" />
```

**Step 2:** Define content separately with `<DropdownRef>`

```jsx

  Multiple sections, code blocks, etc.

<DropdownRef type="drop" id="python-ref">
```

## ‚ö†Ô∏è Important Limitations

### React Components Inside Dropdown

**React components (like Tabs) may not work reliably inside dropdowns** due to how content is rendered.

<Dropdown stripCss defaultValue="tabs-inline" placeholder="Select example..." client:load>
  <DropdownOption value="tabs-inline" label="Tabs Inside (May Not Work)">
    <Tabs defaultValue="tab1" client:load>
      <TabItem value="tab1" label="Tab 1">
        ## First Tab
        This may not work properly after page reload.
      </TabItem>
      <TabItem value="tab2" label="Tab 2">
        ## Second Tab
        React interactivity might be lost.
      </TabItem>
    </Tabs>
  </DropdownOption>
  <DropdownOption value="markdown" label="Markdown (Works Fine)">
    ## Regular Markdown Content
    
    Static Markdown content works perfectly:
    
    - Lists
    - **Bold text**
    - `Code snippets`
    
    ```js
    console.log("Code blocks work great!");
    ```
  </DropdownOption>
</Dropdown>

**What Works:**
- ‚úÖ Markdown (text, lists, tables)
- ‚úÖ Code blocks with syntax highlighting
- ‚úÖ Images, links, quotes
- ‚úÖ Static HTML content

**What May Not Work:**
- ‚ö†Ô∏è React Tabs component
- ‚ö†Ô∏è Custom React components with state
- ‚ö†Ô∏è Interactive elements requiring event handlers
- ‚ö†Ô∏è Any component using `client:load` inside dropdown

**Recommendation:** Use dropdowns primarily for static Markdown content and code examples. For complex interactive content, consider alternative layouts.
