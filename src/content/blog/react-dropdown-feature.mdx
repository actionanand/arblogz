# Complete Guide to Dynamic Dropdowns in Astro MDX

Create powerful, interactive dropdowns in your Astro MDX files that dynamically change content based on user selection. Perfect for language translations, code examples in different frameworks, or any scenario where you need to show different content based on user choice.

    ## JavaScript: Getting Started with Variables
    
    JavaScript uses `let`, `const`, and `var` to declare variables:
    
    ```js title="variables.js" showLineNumbers
    // Modern way: use const for values that don't change
    const apiKey = "your-api-key-here";
    const maxRetries = 3;
    
    // Use let for values that will change
    let currentCount = 0;
    let isLoading = false;
    
    // Avoid var in modern JavaScript
    // var is function-scoped and can cause bugs
    
    // Template literals for string interpolation
    const greeting = `Hello, the count is ${currentCount}`;
    console.log(greeting);
    
    // Arrow functions
    const increment = () => {
      currentCount++;
      console.log(`New count: ${currentCount}`);
    };
    
    increment(); // Output: New count: 1
    ```
    
    ### Key Takeaways
    
    - Use `const` by default
    - Use `let` when you need to reassign
    - Avoid `var` in modern code
    - Template literals make string formatting easier

    ## Building a Counter in React
    
    React uses hooks for state management:
    
    ```jsx title="Counter.jsx" showLineNumbers
    import React, { useState } from 'react';
    
    function Counter() {
      // useState hook for state management
      const [count, setCount] = useState(0);
      const [step, setStep] = useState(1);
      
      const increment = () => {
        setCount(prevCount => prevCount + step);
      };
      
      const decrement = () => {
        setCount(prevCount => prevCount - step);
      };
      
      const reset = () => {
        setCount(0);
      };
      
      return (
        <div className="counter">
          <h2>Count: {count}</h2>
          
          <div className="controls">
            <button onClick={decrement}>-{step}</button>
            <button onClick={reset}>Reset</button>
            <button onClick={increment}>+{step}</button>
          </div>
          
          <div className="step-control">
            <label>
              Step:
              <input
                type="number"
                value={step}
                onChange={(e) => setStep(Number(e.target.value))}
                min="1"
              />
            </label>
          </div>
        </div>
      );
    }
    
    export default Counter;
    ```
    
    ### React Approach
    
    - **Hooks**: `useState` for local state
    - **Declarative**: Describe what UI should look like
    - **Functional**: Modern React uses function components
    - **Immutable**: State updates create new values

    ## REST API Authentication
    
    Traditional REST API with token authentication:
    
    ```js title="rest-api.js" showLineNumbers
    // REST API Configuration
    const API_BASE_URL = 'https://api.example.com/v1';
    const AUTH_TOKEN = 'your-auth-token-here';
    
    // GET request
    async function getUsers() {
      const response = await fetch(`${API_BASE_URL}/users`, {
        method: 'GET',
        headers: {
          'Authorization': `Bearer ${AUTH_TOKEN}`,
          'Content-Type': 'application/json'
        }
      });
      
      if (!response.ok) {
        throw new Error(`HTTP error! status: ${response.status}`);
      }
      
      return await response.json();
    }
    
    // POST request
    async function createUser(userData) {
      const response = await fetch(`${API_BASE_URL}/users`, {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${AUTH_TOKEN}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify(userData)
      });
      
      return await response.json();
    }
    
    // Usage
    const users = await getUsers();
    console.log('Users:', users);
    ```
    
    ### REST API Characteristics
    
    - **Stateless**: Each request contains all needed info
    - **Resource-based**: URLs represent resources
    - **HTTP methods**: GET, POST, PUT, DELETE, PATCH
    - **Token auth**: Bearer tokens in headers

    ## tRPC End-to-End Type Safety
    
    Full-stack TypeScript with no code generation:
    
    ```ts title="trpc-api.ts" showLineNumbers

    import { initTRPC } from '@trpc/server';
    import { z } from 'zod';
    
    const t = initTRPC.create();
    
    export const appRouter = t.router({
      getUsers: t.procedure
        .query(async () => {
          // Your database logic here
          const users = await db.users.findMany();
          return users;
        }),
      
      createUser: t.procedure
        .input(z.object({
          name: z.string(),
          email: z.string().email(),
          isActive: z.boolean().optional()
        }))
        .mutation(async ({ input }) => {
          const user = await db.users.create({
            data: input
          });
          return user;
        }),
      
      getUserById: t.procedure
        .input(z.number())
        .query(async ({ input }) => {
          const user = await db.users.findUnique({
            where: { id: input }
          });
          return user;
        })
    });
    
    export type AppRouter = typeof appRouter;
    
    // Client-side usage (fully typed!)
    import { createTRPCProxyClient, httpBatchLink } from '@trpc/client';
    import type { AppRouter } from './server';
    
    const client = createTRPCProxyClient<AppRouter>({

          url: 'http://localhost:3000/trpc',

          }
        })
      ]
    });
    
    // Usage with full type safety
    const users = await client.getUsers.query();
    //    ^? User[] - TypeScript knows the exact type!
    ```

    Protocol Buffers with efficient binary serialization:
    
    ```proto title="user.proto"

    package user;

    }
    
    // Message definitions
    message Empty {}
    
    message User {
      int32 id = 1;
      string name = 2;
      string email = 3;
      bool is_active = 4;
      int64 created_at = 5;
    }
    
    message UserList {
      repeated User users = 1;
    }
    
    message UserIdRequest {
      int32 id = 1;
    }
    
    message CreateUserRequest {
      string name = 1;
      string email = 2;
      bool is_active = 3;
    }
    
    message UpdateUserRequest {
      int32 id = 1;
      string name = 2;
      string email = 3;
      bool is_active = 4;
    ```
